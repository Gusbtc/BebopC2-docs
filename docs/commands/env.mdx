# env

## Usage

```text
[BEBOP ~]$ env
```

## How it works

Windows walks the process environment block through `GetEnvironmentStringsA` and releases it with `FreeEnvironmentStringsA`. Linux iterates `environ`.

## Output

```text
COMPUTERNAME=DC
USERNAME=Administrator
USERDOMAIN=SUPPORT
PATH=C:\Windows\system32;C:\Windows;...
```

Linux output uses the same `KEY=value` shape.
