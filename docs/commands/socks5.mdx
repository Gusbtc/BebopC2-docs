# socks5

## Usage

```text
[BEBOP ~]$ socks5 start [port]
[BEBOP ~]$ socks5 stop
```

## How it works

SOCKS5 requires session mode. The teamserver listens locally, translates client channels into SOCKS task messages, and the beacon relays target-side traffic. Windows uses dynamically resolved Winsock helpers; Linux uses POSIX sockets and pthread-based relay workers.

## Output

The teamserver allocates a local SOCKS port, defaulting through `1080-1099` when no port is supplied. Output is generated by the teamserver/UI, while channel data is relayed by SOCKS task types.
