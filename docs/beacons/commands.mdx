# Commands

Beacon command support depends on platform and build. The current terminal `help` output is the source of truth for the selected beacon. Output shapes are documented in the [command reference](./command-reference).

## Identity / Target

| Command | Purpose |
| --- | --- |
| `whoami` / `id` | Current user context |
| `hostname` | Hostname |
| `domain` | Joined domain or workgroup context |
| `getpid` | Current process ID |
| `getintegrity` | Windows integrity level |
| `sysinfo` | OS, architecture, memory, hostname, user |
| `drives` | Windows logical drives |
| `uptime` | System uptime |

## Filesystem

| Command | Purpose |
| --- | --- |
| `pwd` | Current directory |
| `cd` | Change directory |
| `ls` / `dir` | List directory |
| `cat` | Read file content |
| `stat` | File metadata |
| `mkdir` | Create directory |
| `rm` | Delete file |
| `rmdir` | Remove empty directory |
| `cp` | Copy file |
| `mv` | Move or rename |

Windows file-browser output is JSON-shaped so the operator UI can render entries, attributes, sizes, and timestamps without parsing terminal tables.

## Linux Utility Set

Linux beacons include a smaller native set:

| Command | Purpose |
| --- | --- |
| `whoami` / `id` | User and group context |
| `hostname` | Hostname and kernel |
| `pwd`, `cd`, `ls`, `cat` | Filesystem operations |
| `mkdir`, `rm`, `cp`, `mv`, `chmod` | Filesystem mutation |
| `ps`, `kill` | Process list and termination |
| `ipconfig` / `ifconfig`, `netstat`, `portscan` | Network inspection |
| `curl` | Plain HTTP request helper |
| `triagedirectory` | Directory summary helper |
| `ssh` | Execute through target host's `ssh` client |
| `download`, `upload`, `sleep`, `exit` | Transfer and control |

Linux `curl` is a lightweight HTTP helper. HTTPS returns a clear unsupported message in that built-in path.

## Result handling

Results are correlated by task label. Empty success is valid for some native WinAPI commands. Large outputs may be truncated or fragmented depending on task type and implementation. For long-running work, prefer session mode or file transfer paths.
