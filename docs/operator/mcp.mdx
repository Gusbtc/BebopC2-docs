# MCP

BebopC2 exposes MCP directly from the teamserver over Streamable HTTP:

```text
POST /api/mcp
Authorization: Bearer <mcp-token>
Content-Type: application/json
```

The agent host only needs network access to the teamserver MCP endpoint and the token shown in the operator UI.

## Operator panel

Open the MCP tab in the operator UI. It shows:

| Field | Meaning |
| --- | --- |
| `mode` | Always `http`. |
| `endpoint` | Full MCP HTTP URL, usually `http://<teamserver>:8080/api/mcp`. |
| `token` | Masked by default; click reveal to copy it. |
| `mutation` | Whether mutating MCP tools are enabled on the teamserver. |

The copy prompt uses endpoint and header values only. It does not depend on a local environment variable, local binary, stdio server, or project checkout.

Generic JSON shape:

```json
{
  "mcpServers": {
    "bebop": {
      "type": "http",
      "url": "http://<teamserver>:8080/api/mcp",
      "headers": {
        "Authorization": "Bearer <mcp-token>"
      }
    }
  }
}
```

Generic TOML shape:

```toml
[mcp_servers.bebop]
url = "http://<teamserver>:8080/api/mcp"
http_headers = { Authorization = "Bearer <mcp-token>" }
```

After configuration, verify the connection with `initialize`, `tools/list`, `resources/list`, and `bebop.sessions.list` before using mutating tools.

## Tools

Read-only tools expose sessions, task results, operator state, and module inventory:

| Tool | Purpose |
| --- | --- |
| `bebop.sessions.list` | List beacon sessions. |
| `bebop.session.get` | Read one beacon session. |
| `bebop.results.list` | Read task results for a beacon. |
| `bebop.task.get` | Read one task result by beacon ID and label. |
| `bebop.task.wait` | Poll until one task result arrives. |
| `bebop.events.list` | Read operator event log entries. |
| `bebop.loot.list` | List exfiltrated loot metadata. |
| `bebop.loot.get` | Read loot content as text or base64. |
| `bebop.library.list` | List BOF and assembly library entries. |
| `bebop.assembly.list` | List uploaded .NET assemblies. |
| `bebop.listeners.list` | List listener definitions. |
| `bebop.socks.list` | List SOCKS proxies. |
| `bebop.terminal.get` | Read terminal and file-browser cache. |
| `bebop.filebrowser.cache.get` | Read only the file-browser cache. |
| `bebop.chat.list` | List operator chat messages. |
| `bebop.commands.list` | List the command catalog. |
| `bebop.command.describe` | Describe one command schema. |

Mutating tools are exposed by default. To force read-only MCP access, start the teamserver with:

```bash
BEBOP_MCP_ALLOW_MUTATION=0
```

With that variable set to `0`, `false`, `no`, or `off`, `tools/list` hides mutating tools and `tools/call` returns `mutation disabled` for mutating tool names.

`bebop.command.run`, `bebop.bof.execute`, and `bebop.inline-assembly.execute` accept structured `args` arrays. The MCP endpoint quotes and packs those arguments before calling the existing teamserver API.

## Schemas

Tool inputs use JSON schemas with `additionalProperties: false`.

| Field | Used by | Meaning |
| --- | --- | --- |
| `beacon_id` | Session, result, and mutating tools | Target beacon ID. |
| `args` | Command, BOF, and inline assembly tools | Ordered argument array. Defaults to `[]`. |
| `transport` | `bebop.command.run` | `auto`, `http`, or `session`; `auto` uses session transport. |
| `mode` | `bebop.inline-assembly.execute` | `auto` or `bridge`; direct mode is disabled. |
| `jitter` | `bebop.beacon.sleep` | `0` through `100`. |
| `port` | `bebop.socks.start` | SOCKS port, or `0` for automatic selection. Requires `bebop.beacon.interactive` first. |
| `confirm` | Destructive tools | Must be `true` for beacon exit, listener delete, library delete, assembly delete, and loot delete. |
| `content_base64` | Upload/build-related tools | Base64 input or output for binary payloads. |

MCP resources expose read-only data as JSON:

| Resource | Purpose |
| --- | --- |
| `bebop://sessions` | All sessions. |
| `bebop://sessions/{id}` | One session. |
| `bebop://results/{id}` | Results for one beacon. |
| `bebop://events` | Event log. |
| `bebop://loot` | Loot metadata. |
| `bebop://library` | Library entries. |
| `bebop://assemblies` | Uploaded assemblies. |
| `bebop://listeners` | Listener definitions. |
| `bebop://socks` | SOCKS proxies. |
| `bebop://terminal/{id}` | Saved terminal state. |
| `bebop://filebrowser/{id}` | Saved file-browser cache. |
| `bebop://filebrowser-session/{id}` | Saved session file-browser cache. |
| `bebop://chat` | Operator chat history. |

## Safety

MCP uses bearer-token auth. The `/api/mcp` endpoint accepts normal operator JWTs and the configured MCP token.

Mutating actions go through the existing teamserver routes, so they keep the same task validation and event logging behavior.

The token file supports either a plain token, which keeps full compatibility, or a scoped format:

```text
token=<secret>
scopes=read,task,file,library,listener,build,admin
```

Destructive tools require `confirm=true` in addition to the required scope.
